⚡ Claude Token-Effizienz Checkliste

Praktische Tipps für Mai 2026 — sinnvoll arbeiten, Token nicht verschwenden

🎯 Ziel: Diese Checkliste zeigt dir konkrete Maßnahmen, um mit Claude effizienter zu arbeiten und unnötige Kosten zu sparen. Kategorisiert nach Impact und Use-Case.

💡 Tipp: Öffne die Kategorien, durchlaufe sie, und setz Checkboxen bei umgesetzten Maßnahmen. So entsteht dein persönliches Token-Spar-Profil.

🤖 1. Richtige Modellwahl HÖCHST WICHTIG

Haiku für einfache Aufgaben nutzen

Zusammenfassungen, Übersetzungen, einfache Formatierungen, Datenextraktion aus strukturierten Text → immer Haiku verwenden.

💰 Spart ~80% Kosten gegenüber Opus

Sonnet für moderate Aufgaben

Textentwürfe, Code-Snippets, mittelständige Analyse, kreative Inhalte → Sonnet ist das Arbeitstier für den Alltag.

💰 Gutes Preis-Leistungs-Verhältnis (~20% von Opus)

Opus nur für komplexe Probleme

Mehrstufenlogik, tiefes Reasoning, komplexe Architekturentscheidungen → wirklich nur dann nutzen, wenn die anderen Modelle nicht reichen.

💰 Nutze die erste Antwort von Sonnet, bevor du zu Opus greifst
📏 2. Kontextgröße & Wiederholungen vermeiden HÖCHST WICHTIG

Nur relevante Dokumente hochladen

Nicht die ganze Datei hochladen — kopiere nur die relevanten Absätze oder Seiten. Jedes zusätzliche Wort kostet Tokens.

💰 50 Seite einer PDF hochladen statt 500

Wiederholten Kontext auslagern (Projekte)

Wenn du denselben Kontext (Codebase, Richtlinien, Anweisungen) in mehreren Chats nutzt → in ein Claude-Projekt als Dokument ablegen und per Referenz einbinden.

💰 Erspart Dir 1000+ Tokens pro Chat

Lange Konversationen segmentieren

Wenn ein Chat über 30–40 Turns wird → neuen Chat starten. Claude kann dann mit frischem Kopf ansetzen, und die alte Konversationshistorie kostet keine Tokens mehr.

💰 Verhindert exponentielles Token-Wachstum

System Prompts (Anweisungen) in Projekte

Wiederkehrende Anweisungen (dein Schreibstil, Formatierungsregeln, spezifisches Kontext-Setup) → einmalig im Projekt als System Prompt setzen, nicht in jedem Turn wiederholen.

💰 Spart hunderte von Tokens pro Chat
✍️ 3. Prompt-Effizienz WICHTIG

Fülltexte reduzieren

Statt „Könntest du bitte vielleicht..." → „Erstelle [X]". Direkt, präzise, kein Höflichkeitsfett.

💰 Spart 10–15% Kontext-Overhead

Mehrere kleine Fragen bündeln

Statt 3 separate Prompts → eine Frage mit 3 Unterpunkten stellen. Claude verarbeitet es effizienter.

💰 Spart ~25% durch Konsolidierung

Längenangaben geben

„Antworte in max. 3 Sätzen" oder „nur die Schlüsselpunkte" steuert die Antwortlänge und spart Tokens bei der Ausgabe.

💰 Verhindert unnötig lange Antworten

Formatierungsvorgaben konkret machen

Statt „formatiere schön" → „als Markdown-Tabelle mit 4 Spalten" oder „JSON mit folgender Struktur" → Claude generiert exakt das Gewollte ohne Umweg.

💰 Spart Iterationen & Umformatierungen
📝 4. Antwortlängen-Kontrolle WICHTIG

Output Tokens bewusst steuern

Input-Tokens sind oft eine Seite, Output kann aber 3x oder 10x größer werden, wenn Claude keine Grenzen hat. Setze ein Ziel: „max. 200 Wörter" oder „Bullet Points".

💰 Kann die Kosten einer Antwort halbieren

Explizite Strukturvorgaben nutzen

Statt freien Text → „Antworte nur als JSON mit Key: Value" oder „3 Bullets, je max 30 Wörter" reduziert Overhead und Speichulation.

💰 Spart 20–30% der Output-Tokens

Beispiel-Output vorgeben

Statt zu beschreiben, was du brauchst → ein echtes Beispiel hinzufügen. Claude orientiert sich an der Länge und Struktur des Beispiels.

💰 Effektiver als lange Beschreibungen
🔄 5. Iterationsschleifen optimieren WICHTIG

Klare Anforderungen upfront

Je unklar die erste Anfrage, desto mehr Iterationen brauchst du. 5 Minuten extra für einen präzisen Prompt spart 10 Umformulierungen.

💰 Spart das Zehnfache an Tokens

Optionen statt Iterationen

Statt Claude zu sagen „das ist falsch, versuch nochmal" → „zeige mir 2–3 Optionen, dann wähle ich". Schneller, klarer, Token-effizienter.

💰 Braucht oft nur eine Frage statt drei

Frühe Feedback-Schleifen

Nach einer Antwort: kurzes Feedback statt Neustart. „Zu lang → kürzen" ist billiger als „vergiss das, fang neu an".

💰 Nachsteuerung ist günstiger als Neustart
📁 6. Projekte & persistente Ressourcen nutzen EMPFOHLEN

Wiederverwendbare Vorlagen als Projektdokumente

Deine Standard-Codebase, Designrichtlinien, Stil-Guides → einmalig in ein Projekt hochladen, in jedem neuen Chat via Referenz einbinden.

💰 Kumulatives Sparpotenzial sehr hoch

Projektanweisungen für wiederkehrende Konventionen

„Schreib immer in Markdown", „nutze itonic-Branding", „verwende Tahoma-Font" → in die Projektanweisungen, nicht in jeden Chat tippen.

💰 Spart hunderte von Tokens über Zeit

Archive abgelaufener Projekte

Alte Chats & Projekte, die du nicht mehr brauchst → archivieren. Streamlined Zugriff, weniger Ablenkung, schnellere Navigation.

💰 Keine Token-Ersparnis, aber bessere Produktivität
🚀 7. Advanced: Batch Processing & API FÜR POWER-USER

Batch API für wiederkehrende Jobs

Wenn du täglich 100+ ähnliche Prompts abschickst → Batch API nutzen (API). 50% Rabatt, aber asynchron (nicht live).

💰 50% Kostenreduktion für Batch-Jobs

Prompt Caching für häufige Kontexte

Bei der API: Längere, stabile Kontexte (z.B. deine ganze Codebase) mit Prompt Caching speichern → wird zwischen Requests wiederverwendet (90% Kostenersparnis!).

💰 Bis zu 90% Einsparung bei wiederholten Kontexten

Claude Code für größere Aufgaben

Complex Entwicklung → Claude Code nutzen. Dort kannst du iterativ arbeiten, und die Costs sind transparenter pro Projekt verwaltet.

💰 Bessere Übersicht, weniger Umgang
📊 8. Monitoring & gute Gewohnheiten WICHTIG

Periodische Token-Audits

Einmal pro Monat deine 3–4 größten Chats ansehen → waren die Modellwahl und Kontextgröße sinnvoll? Was hättest du anders machen können?

💰 Lernen = kontinuierliche Optimierung

„Copy & Paste"-Vorlagen für Standard-Prompts

Deine häufigsten Prompts → als Snippets speichern (z.B. in Obsidian, OneNote, Snippet-Manager). Schneller, konsistenter, Token-sparend.

💰 Keine Extra-Kosten, aber schneller

Dokumentiere deine Learnings

Nach jedem größeren Projekt: „Was hätte ich Token-technisch besser machen können?" Notieren, ins nächste Projekt übertragen.

💰 Feedback-Loop = beste Optimierung
