Commands
Terminal & Slash-Kommandos
Terminal
claude
Interaktive REPL-Session starten im aktuellen Verzeichnis
claude "prompt"
Einmaligen Task direkt ausführen ohne interaktive Session
claude -p "prompt"
Pipe-Modus: Output direkt in Terminal/Script weiterverwenden
claude --continue
Letzte Session fortsetzen — Kontext bleibt erhalten
claude --resume
Session aus Liste auswählen und fortsetzen
Slash-Kommandos (in Session)
/clear
Kontext zurücksetzen — neues Thema starten
/compact
Kontext komprimieren — Token sparen bei langen Sessions
/doctor
Konfiguration und Umgebung prüfen
/cost
Token-Verbrauch der aktuellen Session anzeigen
CLAUDE.md im Projekt-Root = automatischer Stack-Anker. Wird bei jedem Start geladen — kein manuelles Briefing nötig.

Subagents
Parallele & spezialisierte Agenten
Konzept
Task Tool
Claude Code startet Subagenten via Task-Tool — parallele Ausführung mehrerer unabhängiger Aufgaben in eigenen Subshells.
Isolation
Jeder Subagent hat eigenen Kontext, eigene Dateizugänge — kein geteilter State zwischen Agenten.
Anwendungsfälle itonic
Multi-Repo Review
Frontend + Backend gleichzeitig reviewen — je ein Agent pro Repo.
bbs-net
Test-Pipeline
Vitest (Frontend) und pytest (Backend) parallel ausführen, Ergebnisse zusammenführen.
Vitest
pytest
Code + Dok parallel
Agent 1 generiert Code, Agent 2 erstellt parallel die .docx-Dokumentation.
docx
n8n Workflow-Agent
Claude API in n8n als Subagent — Workflows automatisieren via Anthropic-Node.
n8n
Best Practice: Subagenten brauchen eigene CLAUDE.md oder klare System-Prompts — sonst driften sie vom Stack ab.

Hooks
Automatische Lifecycle-Aktionen
Was sind Hooks?
Konzept
Shell-Befehle, die Claude Code automatisch an definierten Punkten im Workflow ausführt — unabhängig vom LLM.
Konfiguration
Definiert in settings.json (global) oder CLAUDE.md (projekt-lokal) unter dem hooks-Schlüssel.
Hook-Events
PreToolUse PostToolUse Notification Stop
PreToolUse
Vor jedem Tool-Aufruf — z.B. Linting vor dem Schreiben einer Datei, Validierung, Logging.
PostToolUse
Nach jedem Tool-Aufruf — z.B. Auto-Format nach Dateiänderung, Tests triggern, Git-Stage.
Stop
Wenn Claude fertig ist — z.B. docker compose up --build automatisch starten, Benachrichtigung senden.
itonic Hook-Rezepte
Auto-Format on Write
PostToolUse → npx prettier --write bei .ts/.tsx Dateien
Frontend
pytest on Save
PostToolUse → pytest -x --tb=short bei Python-Änderungen
Backend
Auto-Deploy on Stop
Stop → docker compose up --build -d auf bbs-net
Docker
Hooks laufen deterministisch — kein LLM, kein Zufall. Ideal für Qualitätssicherung und Deployment-Automatisierung im itonic Stack.
